사용법 :
기본 주소 : localhost:5001/api/room/
항목 조회(모두) : localhost:5001/api/room/[테이블 이름]
예시 : localhost:5001/api/room/equipment
항목 조회(단일) : localhost:5001/api/room/[테이블 이름]/[기본키]
예시 : localhost:5001/api/room/building/15
항목 조회(조건문) : localhost:5001/api/room/[테이블 이름]_condt -> POST 방식 { "condt" : "조건문 "}
예시 : localhost:5001/api/room/equipment_condt + { "condt" : "price>300 AND lifeYear<15"}
항목 추가 : localhost:5001/api/room/[테이블 이름]_add -> POST 방식 {"항목명" : 내용 }
예시 : localhost:5001/api/room/building_add + { "buildingNo" : 1, "buildingName" : "새로운 건물", "totalFloor" : 1 } (json형태의 데이터로 전달해야됨)
항목 제거 : localhost:5001/api/room/[테이블 이름]_delete -> POST 방식 {"기본키" : 내용}
예시 : localhost:5001/api/room/building_delete + { "buildingNo" : 1 } (기본키 buildingNo가 1인 항목이 제거됨)
항목 업데이트 : localhost:5001/api/room/[테이블 이름]_update -> POST 방식 {"기본키" : 내용, "수정할 항목" : "수정할 내용, "수정할 항목" : "수정할 내용"...}
예시 : localhost:5001/api/room/building_update + { "buildingNo" : 1, "buildingName" : "수정된 건물", "totalFloor" : 3 } (buildingNo가 1인 항목의 내용이 수정됨)
